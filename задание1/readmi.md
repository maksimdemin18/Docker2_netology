Docker Compose нужен для того, чтобы описывать и запускать сразу несколько контейнеров как единую систему с помощью одного файла docker-compose.yml: базы данных, бекенд, фронтенд, очереди и т.д. 
Вместо того чтобы вручную писать длинные команды docker run и помнить настройки, один раз описываем всё в конфиге, а потом управляем проектом, командами вроде docker compose up -d. 
Это упрощает жизнь, так как можно быстро поднимать одинаковые окружения на разных машинах (домашний ПК, рабочий ноутбук, тестовый сервер), не бояться «сломать» систему и легко делиться конфигом с другими — достаточно отправить один файл, и у всех будет такой же стенд.
